# Framework Name

Conflict Resolution Diagram

## Core Mental Model

Conflicts persist because parties optimize for competing needs under shared constraints. Better solutions come from surfacing the underlying needs and breaking false tradeoffs.

## When To Use

- Two goals appear to conflict.
- Stakeholders are locked in opposition.
- You need a win-win path instead of a compromise by exhaustion.

## When NOT To Use

- There is no real conflict, only unclear priorities.
- The task is broad creative ideation.
- The issue is causal diagnosis rather than constraint conflict.

## Key Diagnostic Questions

- What are the two conflicting actions or positions?
- What need is each side trying to meet?
- What shared objective sits above both?
- Which assumptions make the conflict seem unavoidable?

## Step-by-Step Reasoning Process

1. State the two conflicting actions or positions.
2. Identify the need each action tries to satisfy.
3. Identify the shared goal above both needs.
4. Surface the assumptions that make the conflict feel necessary.
5. Challenge or remove weak assumptions.
6. Generate options that satisfy both needs better than the current tradeoff.
7. Return the conflict map and best integrative option.

## Example Application

If product wants to ship fast and engineering wants to reduce risk, identify the shared goal of delivering customer value sustainably, then explore changes that satisfy both speed and reliability needs.

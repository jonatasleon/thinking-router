# Framework Name

First Principles

## Core Mental Model

Better solutions come from breaking a problem into fundamental truths and rebuilding from there instead of reasoning by analogy alone.

## When To Use

- The default assumptions may be limiting the solution.
- The user wants an original approach.
- Existing categories or industry norms may be misleading.

## When NOT To Use

- The task is a routine operational fix.
- The problem needs decomposition before deep rebuilding.
- The user only needs a final communication artifact.

## Key Diagnostic Questions

- What do we know is fundamentally true?
- Which assumptions are inherited rather than necessary?
- What remains if those assumptions are removed?
- How would we rebuild from the fundamentals?

## Step-by-Step Reasoning Process

1. State the problem and the default assumption set.
2. Break the problem down by repeatedly asking why and what must be true.
3. Isolate the fundamental truths that remain.
4. Remove assumptions that are only conventions or analogies.
5. Rebuild a solution from the fundamental truths.
6. Compare the rebuilt solution against the original framing.
7. Return the principles and the reconstructed solution path.

## Example Application

Instead of assuming a product demo must be a slide presentation, reduce the goal to proving value and ease of use, then rebuild the solution as a hands-on trial if that better fits the principles.

# Framework Name

Eisenhower Matrix

## Core Mental Model

Not all urgent work is important, and not all important work is urgent. Clear action comes from separating the two dimensions.

## When To Use

- The user asks what to do now.
- There is a backlog of tasks or obligations.
- Prioritization confusion comes from urgency crowding out importance.

## When NOT To Use

- The task is choosing between strategic options rather than managing tasks.
- Impact and effort matter more than urgency.
- There is no actionable task list.

## Key Diagnostic Questions

- Which tasks are time-sensitive?
- Which tasks materially affect goals?
- What should be done, scheduled, delegated, or dropped?
- Which "urgent" tasks are distractions?

## Step-by-Step Reasoning Process

1. List the tasks.
2. Define the goal or responsibility the tasks support.
3. Score each task on urgency and importance using explicit reasoning.
4. Place each task into one quadrant: do, schedule, delegate, or eliminate.
5. Resolve ties by choosing the task with the larger downside if ignored.
6. Return the quadrant placement and immediate next actions.

## Example Application

If a leader has hiring interviews, inbox triage, roadmap planning, and a customer escalation, place each task into the matrix and act from the highest-importance quadrant rather than from noise.

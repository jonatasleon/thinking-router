# Framework Name

Balancing Feedback Loop

## Core Mental Model

Some systems push back against change. A balancing loop stabilizes the system by counteracting movement away from a target or norm.

## When To Use

- The system resists change or returns to equilibrium.
- Gains are temporary and then flatten or reverse.
- A user asks why improvement efforts stall.

## When NOT To Use

- Growth is compounding rather than self-correcting.
- You have not yet identified the relevant variables.
- The issue is mainly structural mapping rather than dynamic behavior.

## Key Diagnostic Questions

- What variable is trying to change?
- What response pushes it back?
- What target, threshold, or norm is being defended?
- Where is delay hiding the stabilizing effect?

## Step-by-Step Reasoning Process

1. Name the variable that changes.
2. Identify the response that counteracts that change.
3. Describe the loop in a causal sentence chain.
4. Identify the target state or norm implied by the loop.
5. Note any delays that make the loop hard to notice.
6. Explain how the loop stabilizes the system.
7. Recommend how to work with, bypass, or redesign the stabilizing mechanism.

## Example Application

If support hires temporarily reduce backlog but new ticket volume rises until the team is overloaded again, identify the mechanism that keeps the system near its prior state.

# Framework Name

Confidence Determines Speed vs. Quality

## Core Mental Model

Execution speed should match confidence. High confidence supports speed; low confidence requires more validation to protect quality.

## When To Use

- The user is trading off speed and quality.
- A team must decide how much rigor to apply before shipping.
- Confidence in the plan varies across work items.

## When NOT To Use

- The issue is choosing among multiple strategic options.
- Confidence cannot even be estimated.
- Urgency/importance is the real prioritization problem.

## Key Diagnostic Questions

- How confident are we in the problem definition?
- How confident are we in the solution?
- What is the cost of being wrong?
- Where can we move fast safely?

## Step-by-Step Reasoning Process

1. State the action under consideration.
2. Estimate confidence in the understanding of the problem and solution.
3. Estimate downside if quality is insufficient.
4. Place the work on a spectrum from move fast to validate heavily.
5. Define the minimum validation needed before acting.
6. Return the recommended pace and what evidence would increase confidence.

## Example Application

For a cosmetic copy tweak with strong evidence, ship fast. For a pricing change with uncertain behavior and large downside, slow down and validate first.

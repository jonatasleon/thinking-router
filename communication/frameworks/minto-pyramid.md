# Framework Name

Minto Pyramid

## Core Mental Model

Busy audiences understand and act faster when you lead with the conclusion, then support it with key points, then details.

## When To Use

- The user needs a concise summary, recommendation, update, or memo.
- The audience has limited time.
- The message needs top-down clarity.

## When NOT To Use

- The user is giving delicate interpersonal feedback.
- The message needs exploratory discussion rather than conclusion-first delivery.
- The reasoning behind the recommendation is still incomplete.

## Key Diagnostic Questions

- What is the main conclusion?
- What 2-4 key points support it?
- What detail is necessary for credibility?
- What can be omitted without weakening the message?

## Step-by-Step Reasoning Process

1. Write the main recommendation or takeaway first.
2. Group supporting points into 2-4 major reasons.
3. Add only the details required to justify those reasons.
4. Remove information that does not support the conclusion.
5. Check whether the audience can act after reading only the top section.
6. Return the final message in conclusion -> key points -> details order.

## Example Application

Lead with "We should delay launch by one week," then give the main reasons such as unresolved defects and support readiness, then include the specific data only if needed.

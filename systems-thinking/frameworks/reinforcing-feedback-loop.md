# Framework Name

Reinforcing Feedback Loop

## Core Mental Model

Some systems amplify change. A reinforcing loop compounds movement in the same direction and can create runaway growth or decline.

## When To Use

- Growth or decline accelerates over time.
- Small changes become large effects.
- The user asks about compounding or spiral dynamics.

## When NOT To Use

- The system clearly self-corrects toward stability.
- There is no loop, only a one-way chain.
- The main need is concept organization rather than system dynamics.

## Key Diagnostic Questions

- What variable is increasing or decreasing?
- What effect makes that variable move further in the same direction?
- What makes the loop accelerate?
- What constraints might eventually limit it?

## Step-by-Step Reasoning Process

1. Name the variable that is compounding.
2. Identify the effect that feeds back into further increase or decrease.
3. Write the loop as a closed causal chain.
4. Describe why the loop amplifies rather than stabilizes.
5. Note any delays or thresholds.
6. Identify likely risks if the loop continues.
7. Recommend where to dampen, redirect, or exploit the loop.

## Example Application

If a product gets better usage data from more users, which improves the product, which attracts more users, identify the reinforcing growth loop and where it can break.

# Framework Name

Productive Thinking Model

## Core Mental Model

Creative problem-solving works best when you move from understanding, to success definition, to catalytic questions, to idea generation, to solution forging, to resource alignment.

## When To Use

- The problem is defined but the solution is not.
- The user wants an end-to-end creative problem-solving process.
- You need to move from ambiguity to an action plan.

## When NOT To Use

- The task is only root-cause analysis.
- A simple solution is already obvious.
- The user mainly needs a decision between explicit options.

## Key Diagnostic Questions

- What is going on?
- What does success look like?
- What is the real question?
- What candidate answers exist?
- Which solution best meets the success criteria?
- What resources are required?

## Step-by-Step Reasoning Process

1. Ask "What's going on?" and summarize the current reality.
2. Ask "What's success?" and define success criteria, constraints, values, and essential outcomes.
3. Ask "What's the question?" and create catalytic questions.
4. Generate many candidate answers without early judgment.
5. Forge the solution by evaluating ideas against the success criteria.
6. Align resources by turning the chosen solution into a plan.
7. Return the chosen solution, why it wins, and the execution outline.

## Example Application

If the goal is winning a strategic customer, define what success means, identify catalytic questions about value demonstration, generate several solution paths, choose one, and assign owners and resources.

# Framework Name

Iceberg Model

## Core Mental Model

Surface events are outputs of deeper patterns, structures, and mental models. Better interventions come from reasoning below the visible event level.

## When To Use

- A problem keeps recurring after local fixes.
- The user asks why an event keeps happening.
- You need leverage, not just explanation.

## When NOT To Use

- The task is to compare options or prioritize work.
- The user mainly needs a message drafted.
- There is only a single one-off event with no sign of recurrence.

## Key Diagnostic Questions

- What event is visible right now?
- What patterns over time surround it?
- What structures produce those patterns?
- What beliefs, incentives, or assumptions keep the structures in place?

## Step-by-Step Reasoning Process

1. State the event in one sentence.
2. List known facts about the event; separate assumptions.
3. Identify repeated patterns or trends over time.
4. Infer the structures creating those patterns: incentives, workflows, rules, dependencies, or feedback loops.
5. Infer the mental models beneath the structures: beliefs, priorities, norms, or fears.
6. Check that each deeper layer plausibly explains the layer above it.
7. Identify the highest-leverage intervention level.
8. Return the event, pattern, structure, mental model, and recommended intervention.

## Example Application

If every launch ships with bugs, do not stop at "the last release was buggy." Identify the pattern of repeated buggy launches, the structure of late QA and deadline pressure, and the mental model that speed matters more than quality.

# Framework Name

Issue Trees

## Core Mental Model

Large problems become solvable when they are broken into mutually exclusive, collectively exhaustive branches that can be investigated or solved in order.

## When To Use

- The user asks to break a problem down.
- The problem is broad and hard to start.
- You need a systematic structure for investigation.

## When NOT To Use

- The problem is already narrow and concrete.
- The main task is choosing among options.
- The user needs creative recombination more than decomposition.

## Key Diagnostic Questions

- What is the top-level problem?
- What major branches explain or solve it?
- Are the branches MECE?
- Which branch matters most?

## Step-by-Step Reasoning Process

1. Write the top-level problem or question.
2. Choose whether you need a problem tree (why) or solution tree (how).
3. Break the top-level item into broad branches.
4. Test the branches for MECE.
5. Expand only the branches needed to make progress.
6. Use data or evidence to prioritize the highest-value branch.
7. Return the tree and the recommended focus area.

## Example Application

If feature adoption is low, split the problem into users who do not know about the feature and users who know about it but do not use it, then expand the stronger branch.

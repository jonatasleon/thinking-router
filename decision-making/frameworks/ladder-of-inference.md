# Framework Name

Ladder of Inference

## Core Mental Model

People jump from observation to conclusion by selecting data, interpreting it, and building assumptions. Good decisions slow that climb and test each rung.

## When To Use

- The user may be jumping to conclusions.
- A recommendation rests on interpretation-heavy evidence.
- Team conflict comes from different readings of the same event.

## When NOT To Use

- The task is routine prioritization.
- There is abundant hard evidence and little ambiguity.
- The user needs creative ideation, not assumption checking.

## Key Diagnostic Questions

- What did we actually observe?
- What data did we select and ignore?
- What meaning did we add?
- Which assumptions connect the data to the conclusion?
- What else could be true?

## Step-by-Step Reasoning Process

1. State the current conclusion or intended action.
2. Write the raw observable data.
3. Identify which data points were selected as meaningful.
4. State the interpretation applied to those data.
5. Surface the assumptions behind that interpretation.
6. State the resulting conclusion and action.
7. Test alternative interpretations and missing data.
8. Revise the conclusion if the ladder is weak.

## Example Application

If a manager concludes "the team is disengaged" because people were quiet in a meeting, separate the observation from the assumption and consider other explanations such as confusion or lack of preparation.

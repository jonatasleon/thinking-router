# Framework Name

Zwicky Box

## Core Mental Model

Novel solutions emerge when you systematically recombine dimensions of a problem instead of brainstorming from habit.

## When To Use

- The user needs creative solution generation.
- The problem has several adjustable dimensions.
- Existing ideation feels repetitive.

## When NOT To Use

- The main need is root-cause diagnosis.
- The problem is too simple for morphological analysis.
- The dimensions of the problem are still unclear.

## Key Diagnostic Questions

- What are the main dimensions of the problem?
- What possible values exist for each dimension?
- Which combinations have not been explored?
- Which combinations are promising and feasible?

## Step-by-Step Reasoning Process

1. State the design or solution challenge.
2. Identify the core dimensions that define possible solutions.
3. List the possible values for each dimension.
4. Build a matrix of combinations.
5. Scan for combinations that are novel, useful, or overlooked.
6. Discard impossible combinations.
7. Return a short list of the strongest candidates with why they are interesting.

## Example Application

For a support offering, vary delivery channel, response time, level of human involvement, and pricing model to generate solution combinations beyond the current default.

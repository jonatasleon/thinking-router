# Framework Name

Decision Matrix

## Core Mental Model

When options can be compared on shared criteria, weighted scoring makes tradeoffs explicit instead of intuitive and hidden.

## When To Use

- There are multiple options.
- The options can be judged against common criteria.
- The user wants a reasoned winner, not just discussion.

## When NOT To Use

- Options are fundamentally incomparable.
- Criteria are too unclear to score.
- The decision is mainly values-based rather than evaluative.

## Key Diagnostic Questions

- What decision is being made?
- What are the real options?
- Which criteria matter most?
- How should criteria be weighted?
- Does the winner hold up if weights change?

## Step-by-Step Reasoning Process

1. State the decision.
2. List the options.
3. Define the evaluation criteria.
4. Assign weights to the criteria.
5. Score each option against each criterion with brief justification.
6. Multiply score by weight and total the results.
7. Run a quick sensitivity check on the most important weights.
8. Return the ranking, why the top option wins, and what assumptions matter most.

## Example Application

To choose a tool, score options on cost, collaboration, prototyping, and learning curve, then test whether the same winner holds if collaboration matters more than cost.

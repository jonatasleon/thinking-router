# Framework Name

OODA Loop

## Core Mental Model

In fast-changing environments, advantage comes from quick cycles of observing, orienting, deciding, and acting rather than waiting for perfect information.

## When To Use

- The situation is moving quickly.
- Waiting for full certainty is costly.
- You need iterative action under uncertainty.

## When NOT To Use

- The decision is stable and high stakes with time for analysis.
- The environment is not changing materially.
- The main need is structured option comparison.

## Key Diagnostic Questions

- What is happening right now?
- How should we interpret it with current context?
- What is the next best action?
- What will we learn from acting?

## Step-by-Step Reasoning Process

1. Observe the latest facts and signals.
2. Orient by interpreting those facts through goals, constraints, and prior knowledge.
3. Decide the next action, favoring reversibility when uncertainty is high.
4. Act.
5. Capture what changed after the action.
6. Loop again with updated information.
7. Return the current cycle state and the next trigger for review.

## Example Application

During a live incident, gather current symptoms, interpret likely causes, choose the safest stabilizing action, act, then reevaluate with the new signal set.

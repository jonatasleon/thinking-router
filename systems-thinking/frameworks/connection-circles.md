# Framework Name

Connection Circles

## Core Mental Model

Systems become understandable when you map the important elements and the direct cause-and-effect links between them.

## When To Use

- The user asks how a system works.
- Multiple factors influence each other.
- You want to surface feedback loops before proposing changes.

## When NOT To Use

- The system has only one obvious linear cause.
- The task is mainly choosing among options.
- The user needs deep root-cause layering more than relationship mapping.

## Key Diagnostic Questions

- What are the 5-10 key elements in the system?
- Which elements increase or decrease other elements?
- Where do closed loops appear?
- Which links are known facts versus hypotheses?

## Step-by-Step Reasoning Process

1. Define the system boundary.
2. Name up to 10 key elements, using nouns that can increase or decrease.
3. Remove duplicates and overlapping elements.
4. Map direct causal links between elements.
5. Mark each link as increasing or decreasing.
6. Identify closed loops.
7. Classify each loop as probably reinforcing or balancing, if possible.
8. Highlight the few elements with the most outgoing influence or loop participation.
9. Return the element list, key links, loops, and intervention points.

## Example Application

For unhappy customers, map bugs, support tickets, response time, new features, and customer sentiment. The loop between unhappy customers, more tickets, slower response times, and more unhappy customers becomes visible.

# Framework Name

Concept Map

## Core Mental Model

Complex understanding improves when concepts, actors, and dependencies are organized into a structured map rather than a flat list.

## When To Use

- The user needs to understand relationships between ideas or entities.
- You need a map of dependencies, not necessarily feedback loops.
- The task is exploratory and definitional.

## When NOT To Use

- The main task is diagnosing recurrence over time.
- The key issue is dynamic loop behavior.
- The user needs a numerical decision process.

## Key Diagnostic Questions

- What are the central concepts or entities?
- How are they grouped?
- What relationships connect them?
- Which concepts are foundational versus dependent?

## Step-by-Step Reasoning Process

1. State the concept or system being mapped.
2. List the central concepts, actors, or components.
3. Group them into natural clusters.
4. Define the most important relationships between nodes.
5. Mark hierarchy where useful: parent, child, dependency, influence, or example.
6. Check for missing bridge concepts that make the map coherent.
7. Summarize the map in a way that surfaces the main structure.
8. Return the clusters, relationships, and most central nodes.

## Example Application

To explain a product onboarding system, map users, sign-up flow, activation triggers, help content, support, and retention outcomes, then show how they depend on each other.

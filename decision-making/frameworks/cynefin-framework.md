# Framework Name

Cynefin Framework

## Core Mental Model

Different contexts require different decision styles. First classify the situation, then choose the response pattern that fits that domain.

## When To Use

- The user asks how to respond to a situation.
- The context may be clear, complicated, complex, or chaotic.
- You need to avoid using the wrong decision style.

## When NOT To Use

- The decision is simply choosing among options with shared criteria.
- The context is already well classified.
- The user mainly needs interpersonal communication support.

## Key Diagnostic Questions

- Are cause and effect clear?
- Does the situation require expert analysis?
- Are there unknown unknowns?
- Is the situation out of control?
- Are we in disorder because we cannot yet classify it?

## Step-by-Step Reasoning Process

1. State the situation.
2. Test whether it fits clear, complicated, complex, chaotic, or disorder.
3. Choose the matching response pattern:
   - clear -> sense, categorize, respond
   - complicated -> sense, analyze, respond
   - complex -> probe, sense, respond
   - chaotic -> act, sense, respond
   - disorder -> decompose and classify parts
4. Explain why that domain fits better than adjacent ones.
5. Recommend the next action style, not just the next action.

## Example Application

Launching a known operational checklist is clear. Entering a new market with emerging customer behavior is complex and should start with probes rather than full plans.

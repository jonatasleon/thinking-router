# Framework Name

Hard Choice Model

## Core Mental Model

Decision quality improves when you first classify the decision by impact and comparability, then match the amount of effort to the type of choice.

## When To Use

- The user is unsure how much rigor a decision deserves.
- A choice feels difficult but the reason is unclear.
- You need to decide whether to move fast, compare carefully, or accept value conflict.

## When NOT To Use

- The decision is already obviously high-rigor and comparable.
- The task is pure task triage.
- The user already has a scoring framework in progress.

## Key Diagnostic Questions

- How much impact will this decision have?
- How easy is it to compare the options?
- Is this a no-brainer, apples-and-oranges, big choice, or hard choice?
- What level of analysis is appropriate?

## Step-by-Step Reasoning Process

1. State the decision and the main options.
2. Estimate the impact if the decision goes well or badly.
3. Estimate how comparable the options are.
4. Place the decision into one of four types.
5. Choose the matching decision posture:
   - no-brainer -> decide quickly
   - apples-and-oranges -> decide by present values or priorities
   - big choice -> compare carefully
   - hard choice -> accept tradeoffs and consider experiments
6. If needed, hand off to another framework such as Decision Matrix or Second-order Thinking.
7. Return the classification and next decision method.

## Example Application

Choosing between two similar vendors is a big choice. Choosing between startup life and an established company is a hard choice and should not be forced into fake precision too early.
